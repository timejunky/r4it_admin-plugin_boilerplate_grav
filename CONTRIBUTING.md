# Contributing

Thanks for contributing to this Grav Admin boilerplate.

## Support Channels (MIT Edition)

- General support: Grav community channels and this repository's public GitHub issues/discussions.
- This MIT/community edition does not include private 1:1 support.

## Ways to Contribute

- Report bugs
- Propose features
- Improve documentation
- Submit code fixes

## Before Opening an Issue

- Check existing issues first.
- Include reproduction steps and expected behavior.
- Include Grav version, PHP version, and plugin version.
- Include multilingual context if the issue is route-related.

## Pull Request Guidelines

1. Create a focused branch from `main`.
2. Keep PRs small and topic-specific.
3. Update documentation when behavior changes.
4. Add or adapt tests when possible.
5. Ensure coding style remains consistent.

## Local Validation Checklist

- Plugin loads in Grav Admin.
- Sidebar entry appears once and routes correctly.
- Route works with and without language prefix.
- Tabs render correctly (`info`, `settings`, `tools`).

## Commit Message Style

Use clear imperative messages, for example:

- `fix: guard Twig loader registration`
- `docs: add architecture flow details`
- `test: add regression for admin route matching`

## Code of Conduct

Be respectful and constructive. This project aims for practical collaboration and clear technical communication.
